описание проекта:
- сделал две страницы:
	а) список продуктов;
	б) детальная информация по выбранному продукту и коментарии к нему;
- при создании отдельных элиментов использовал библиотеку Material;
- пользователь может авторизироваться на любой странице и разлогиниться так же;
- хранилище глобальных данных (имя пользователя и токен) реализовано в сервисе, 
так как хранить в локальном storege оказалось не целелесообразно ввиду того что 
пользователи и комментарии по истечению какогото времени удаляются.
- приложение адоптирована под мобильные устройства;
- все API запросы работают;

краткое описание структуры проекта:
/project-reviews - проект на angular 8;
/project-reviews/src - папка с основным кодом;
/project-reviews/src/арр - 
/project-reviews/src/арр/components - папка с компоненами: 
	- addcomment - форма добавления сомментариев и рейтинга,
	- authorization - авторизация пользователя,
	- header - кнопки авторизации пользователя и название страницы,
	- list-comments - отображение всех комментариев с рейтингом,
	- messages - пользовательские сообщения.
/project-reviews/src/арр/environment - файл с адресом API
/project-reviews/src/арр/pages - папка со страницами:
	- detail-product - детальная информация о продукте,
	- main - список продуктов.
/project-reviews/src/арр/services - папка с сервисами:
	- api - методы для реализации API
	- datakeep - хранилище данных
	- screen - сервис для определения соотношения сторон 
	на экране устройтсва по нему можно понять это мобильное устройство или декстоп
